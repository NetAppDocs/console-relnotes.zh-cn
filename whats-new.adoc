=== Cloud Volumes ONTAP



==== 能够管理 Cloud Volumes ONTAP 的 Console 代理设置

现在，即使失去连接或代理设置不正确，也可以在 NetApp Console 代理上管理 Cloud Volumes ONTAP 的代理设置。以前，如果 Console 代理在 20 分钟内无法连接到 Cloud Volumes ONTAP，它将使用默认设置覆盖您的手动代理设置。这导致通信失败，包括 AutoSupport 消息问题。若要保留现有系统的代理设置，请运行以下 API 调用：

`PUT /occm/config`

在请求正文中包含以下参数：

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
默认模式为标准模式，这意味着如果 Console 代理在 20 分钟内无法连接到 Cloud Volumes ONTAP，它将使用默认设置覆盖您的代理设置。

https://docs.netapp.com/us-en/console-automation/cvo/put-occm-config.html["配置可修改的 NetApp Console 参数"^]

=== 备份和恢复



==== ONTAP Volumes 工作负载增强

在此版本中，ONTAP Volumes 工作负载引入了以下增强功能：

* *还原到源位置*：您现在可以将文件从本地快照和复制快照还原到源位置以及备用位置。
* *每小时卷发现*：Dashboard 现在可以更快地发现和刷新新卷。
* *支持暗站点*：您现在可以从 https://mysupport.netapp.com/site/products/all/details/netapp-console/downloads-tab/download/61906/4.4.0/downloads["NetApp 支持站点"]下载暗站点的 Backup and Recovery。目前，暗站点的 NetApp Backup and Recovery 仅支持 ONTAP Volumes 工作负载。
* *Unicode 兼容性改进*：目录索引现在可以更好地处理 Unicode 字符。
* *改进的 FlexGroup 兼容性*：将 ONTAP FlexVol 卷转换为 FlexGroup 卷时，现在可以恢复转换前所做的备份。
* *多存储桶支持*：您现在可以使用跨不同云提供商的每个系统最多 6 个存储桶来保护系统中的卷。


要了解有关保护 ONTAP Volume 工作负载的详细信息，请参阅 https://docs.netapp.com/us-en/data-services-backup-recovery/prev-ontap-protect-overview.html["保护 ONTAP Volume 工作负载概述"]。



==== VMware 工作负载增强功能

在此版本中，VMware 工作负载引入了以下增强功能：

* *从对象存储还原访客数据*：您现在可以从对象存储还原访客文件和文件夹。
* *VMFS/NFS 快速还原*：现在可以在还原到备用位置时启用快速还原选项。启用后，将仅从对象存储中检索 VM 元数据（文件、LUN 和命名空间），从而使卷的可用性快于完全恢复。
* *支持 Amazon FSx for NetApp ONTAP*：VMware 工作负载现在支持运行 9.13.1 或更高版本的 Amazon FSx for NetApp ONTAP 存储系统。


要了解有关保护 VMware 工作负载的详细信息，请参阅 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-protect-overview.html["保护 VMware 工作负载概述"]。



=== 数据分类



==== 1.52 版

.资源验证
为了确保可靠性，Data Classification 现在可以在扫描、报告生成和操作中验证可用的内存和磁盘资源。当内存使用率超过 95% 或磁盘空间低于 6 GB 时，Data Classification 会自动暂停操作。一旦资源充足，操作就会恢复。

要监视系统资源，请参见 link:https://docs.netapp.com/us-en/data-services-data-classification/task-health-monitor.html["监控 Data Classification 的运行状况"^]。

.提高了复制和移动的限制
CIFS 目标现在支持复制和移动功能。两个操作的每个文件大小限制已从之前的 50 MB 限制增加到 1 GB。这两个操作的操作限制已增加到 1 TB 或 500 万个文件。

有关详细信息，请参见 link:https://docs.netapp.com/us-en/data-services-data-classification/task-investigate-data.html["调查数据"^]。

.Active Directory 集成配置权限
Data Classification 中对 Active Directory (AD) 集成设置的更改现在仅限于管理员。查看者不再能够修改 AD 设置。

有关 Classification viewer 角色的详细信息，请参见 link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html#data-service-roles["数据服务角色"^]。

.透明代理支持
Data Classification 现在支持透明代理配置，可在代理或受限网络环境中实现更顺畅的部署。

有关在受限环境中部署的详细信息，请参见 link:https://docs.netapp.com/us-en/data-services-data-classification/task-deploy-compliance-dark-site.html["在没有互联网访问权限的 Linux 主机上安装 NetApp Data Classification"^]。

=== Cloud Volumes ONTAP



==== Cloud Volumes ONTAP 支持下一代 Google Cloud 虚拟机

借助 9.18.1，NetApp 将新的 Cloud Volumes ONTAP 部署从 N2 虚拟机过渡到下一代 Google Cloud C3 系列虚拟机，以获得更快、更具可扩展性的体验。现在，您可以在 Google Cloud 中部署 Cloud Volumes ONTAP 9.18.1 及更高版本时利用 C3 系列虚拟机。C3 系列机器通过使用 Google Virtual NIC (gVNIC) 和 Hyperdisk Balanced 磁盘，确保密集型工作负载的动态性能，从而提供更高的性能和更高的容量限制。


NOTE: 目前，Cloud Volumes ONTAP 仅在单节点部署中支持 C3 系列。

如果您的 Cloud Volumes ONTAP 系统运行 9.18.1 或更高版本，则用于轻松部署单节点的预配置包将自动使用 C3 虚拟机，同时使您能够根据工作负载需求自定义 IOPS 和吞吐量参数。同样，在创建聚合时，您可以添加 Hyperdisk Balanced 磁盘，以在 Google Cloud 中实现更好的性能和可扩展性。此外，您可以为默认 Flash Cache 支持选择 C3 系列计算机的 LSSD 变体。

将卷添加到聚合时，无法更改 C3 VM 的磁盘类型，因为 C3 仅支持 Hyperdisk Balanced 磁盘。同样，将具有 N2 VM 类型的系统复制到 C3 VM 时，磁盘类型默认设置为 Hyperdisk Balanced。

https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Google Cloud 中 Cloud Volumes ONTAP 支持的配置"^]

数据分类功能现在支持创建自定义数据类别。您可以上传文件来微调数据分类所使用的 AI 模型，以便将类别标记应用于数据。所有自定义分类的界面都得到了改进。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-data-classification/task-custom-classification.html["创建自定义分类"^]。

.自定义过期数据定义
数据分类功能现在允许您自定义过期数据的定义，以满足您组织的需求。此前，过时数据被定义为三年前最后一次修改的数据。现在，可以根据上次访问时间或上次修改时间来识别过时的数据；时间段可以从 6 个月前到 10 年前不等。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-data-classification/task-stale-data.html["自定义过期数据定义"^]。

.性能提升
数据分类、数据映射报告和调查页面上的筛选器等所有页面的加载时间都已缩短。

.调查报告预计完成时间
下载调查报告时，“数据分类”现在会显示下载完成的预计时间。

=== 控制台设置和管理



==== 控制台代理 4.3.0

4.3.0 版本同时支持标准模式和受限模式。

此版本的控制台代理包含安全改进、错误修复以及以下功能：

.能够验证控制台代理的网络连接
现在，您可以直接从 NetApp Console 验证已连接的 Console 代理的网络连接。此功能有助于验证连接并排除 Console 代理的问题。这是除了从 Console 代理维护控制台命令行界面 (CLI) 运行网络诊断的现有能力之外。

link:https://docs.netapp.com/us-en/console-setup-admin/task-troubleshoot-agent.html#verify-network-access["了解如何从 NetApp Console 运行配置。"]



==== NetApp Console管理

本次发布包含以下内容：

.NetApp Console中联合组的角色访问控制
NetApp Console支持将角色分配给联合组（例如 Active Directory 组），从而允许管理员根据组织身份提供程序 (IdP) 中的组成员身份自动执行用户入职和离职。此功能可减少管理开销，并通过在组成员身份更改时立即更新控制台访问权限来确保安全、一致的访问。

link:https://docs.netapp.com/us-en/console-setup-admin/task-iam-manage-members-permissions.html#add-a-federated-group-to-your-organization["了解如何为您的组织提供对联合组的访问权限。"]

.NetApp Console处于受限模式时支持联合身份验证
现在您可以为处于受限模式的NetApp Console组织启用联合身份验证。这样，用户就可以使用其公司凭证登录控制台，同时保持受限模式的安全优势。

.只读模式
您可以将NetApp Console组织设置为只读模式。在只读模式下，用户可以查看资源和设置，但不能进行任何更改。组织管理员或超级管理员可以启用组织的只读模式。启用只读模式后，具有管理角色的用户必须手动提升权限才能根据需要进行更改。

link:https://docs.netapp.com/us-en/console-setup-admin/task-enable-read-only-mode.html["了解如何为控制台组织启用只读模式。"]

link:https://docs.netapp.com/us-en/console-setup-admin/task-user-settings.html["当你的组织处于只读模式时，了解如何提升你的角色。"]



=== 灾难恢复



==== 支持本地环境中的多个控制台代理

如果您在本地使用灾难恢复，现在可以为每个 vCenter 实例部署控制台代理，从而提高恢复能力。

例如，如果您有两个站点（站点 A 和 B），则站点 A 可以将控制台代理 A 连接到 vCenter 1、 ONTAP部署 1 和ONTAP部署 2。站点 B 可以将控制台代理 B 连接到 vCenter 2 和ONTAP部署 3 和 4。

有关灾难恢复中的控制台代理的信息，请参阅 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/get-started/dr-prerequisites.html#netapp-console-prerequisites["创建控制台代理"^]。



==== 使用基于数据存储的保护机制，在故障转移后为复制计划添加虚拟机

当故障转移被触发时，任何使用基于数据存储的保护的复制计划都会包含已添加到数据存储中的虚拟机，前提是这些虚拟机已被发现。故障转移完成前，您必须提供新增虚拟机的映射详细信息。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/failover.html["故障转移应用程序"^]。



==== 新邮件通知

灾难恢复功能现在会针对以下事件发送电子邮件通知：

* 接近容量使用极限
* 报告生成完成
* 工作失败
* 许可证到期或违规




==== Swagger 改进

现在您可以从灾难恢复中访问 Swagger 文档。在灾难恢复中，选择**设置**，然后选择**API 文档**以链接到 Swagger，或者在浏览器的隐身/隐私模式下访问此 URL： link:https://snapcenter.cloudmanager.cloud.netapp.com/api/api-doc/draas["https://snapcenter.cloudmanager.cloud.netapp.com/api/api-doc/draas"^]。



==== 改进的用户界面

灾难恢复功能现在提供更完善的警告和错误解决方法。此版本修复了导致已取消的作业无法在用户界面中显示的错误。已取消的职位现在可以查看了。当同一个目标网络映射到多个不同的源网络时，还会发出新的警告。



==== 保留复制计划中默认添加的 VM 文件夹结构

创建复制时，新的默认设置是保留 VM 文件夹结构。如果恢复目标没有原始文件夹层次结构，灾难恢复程序会创建它。您可以取消选择此选项以忽略原始文件夹层次结构。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/drplan-create.html["创建复制计划"^]。



=== 勒索软件抵御能力



==== 将快照复制到ONTAP

Ransomware Resilience 现在支持将快照复制添加到辅助 ONTAP 站点。使用复制策略的保护组，您可以为每个工作负载复制到相同的目标或不同的目标。您可以创建包括复制的勒索软件保护策略或使用预定义策略。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-use-protect.html["在勒索软件恢复中保护工作负载"^]。



==== 将工作负载排除在勒索软件恢复能力之外

勒索软件恢复功能现在支持将系统中的特定工作负载从保护范围和勒索软件恢复仪表板中排除。发现工作负载后，您可以将其排除在外；如果想要添加勒索软件防护，则可以重新将其包含在内。排除在外的工作负载无需付费。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-start-discover.html#exclude-workloads["排除工作负载"^]。



==== 标记提醒，如正在审核中

勒索软件恢复功能现在允许您将警报标记为“审核中”。使用“审核中”标签可以提高团队在对活跃的勒索软件威胁进行分类和管理时的清晰度。

有关详细信息，请参阅 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-use-alert.html["管理勒索软件恢复能力中的警报"^]。



=== Cloud Volumes ONTAP



==== Cloud Volumes ONTAP的首选计费方式

现在您可以选择一种首选的计费方式来计算您的Cloud Volumes ONTAP使用量和超额费用。自 2025 年 6 月 25 日起，自带许可证 (BYOL) 许可模式将不再提供， NetApp已在NetApp Console的“许可和订阅”部分添加了首选的计费方式。您可以选择使用年度市场订阅进行计费和超额费用结算，或者选择现有的 BYOL 模式作为首选方案。这样，您可以灵活选择最适合您组织财务战略和使用模式的充电方式。

https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/concept-licensing-charging.html#billing-preferences-and-overages["计费偏好和超额费用"^]。
